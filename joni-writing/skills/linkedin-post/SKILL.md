# LinkedIn Posts: Powerful Ideas (Kraftfulla idéer)

Write LinkedIn posts in Joni Lindgren's tone and style when sharing powerful ideas from work as Product Manager and coach.

## Core Principle

Write about ONE universal insight many can understand – illustrated with concrete examples. Not a case study, not a complete project description, not a lesson. Don't over-explain. Don't give all background. Find the universal idea and tell it simply.

**Length:** up to 20 lines total.

## CRITICAL RULES

### Tone:
- Write in Swedish
- Direct and conversational - like talking to a colleague
- Concrete with numbers, not abstractions
- Humble - acknowledge complexity, don't be a know-it-all
- Practical - what can the reader do with this?

### Language Elements You MUST Use:

- Concrete numbers: "från 23% till 35%", "5 dagar till 2 dagar", "12 till 2 per kvartal"
- Personal markers at least once: "Så här tycker jag:", "In my humble because of experience opinion", "ack så värdefullt det!"
- Emoji bullets 🔵 for points (not too much)
- Definitions with question-mark format: "En Stakeholder Review? Produktteam och stakeholders samlas för att..."
- Rhetorical questions

### Acceptable Meta-Commentary:

You MAY use brief, empathetic meta-commentary that leads directly to examples:

```
"Det är svårt att formulera tydliga Business Outcomes från blankt papper.
Men det är absolut en muskel som kan tränas upp, och när jag coachar PMs
behövs oftast bara ett par konkreta exempel och några omgångar feedback."
```

This is OK because it: (1) acknowledges difficulty, (2) is empathetic, (3) leads directly to example, (4) is brief, (5) not preachy

### ABSOLUTELY FORBIDDEN - NEVER Use:

- "Det här såg jag nyligen..." / "Jag hör ofta..." / "Vi såg att..." / "Folk frågar mig"
- "Jag vill prata om..." / "Idag tänkte jag dela..."
- "När vi coachar team ser vi ofta att..." (long meta-observation)
- "Det här är den vanligaste feedbacken vi ger..."
- Talking ABOUT observing patterns instead of sharing the pattern directly
- Didactic headings: "Vad är problemet?", "Så här gör vi istället:", "Lösningen:"
- Academic language
- Vague generalizations
- Too much explanation of WHY something is a problem (show instead)
- Abstract openings like "Detta mönster dyker upp..."
- Multiple ideas in same post

## OPENING STYLES

### Style 1: Direct into mechanism/solution
Start directly with WHAT happens. No buildup.

**Template:**
```
[Concrete action/mechanism]. Och då [what happens/what we see].
```

**Example:**
```
Deras nya roadmaps presenteras i en Stakeholder Review fyra veckor efter
utbildningen genomförts. Och då vill vi se vägval, prioritering och insikter
som backar upp varför just detta kommer bidra till att de metrics vi bryr oss om.
```

### Style 2: Direct instruction
Say directly what reader should do, followed by rhetorical questions.

**Template:**
```
[Börja/Gör X]. [Retorisk fråga]? [Retorisk fråga]?
```

**Example:**
```
Börja varje roadmapping session med att titta på vilket affärsmål ni jobbat
mot hittills och hur långt ni kommit.

Vilket mätbart affärsresultat har ni jobbat med att flytta? Hur långt har ni kommit?
```

### Style 3: Problem → Solution (brief)
Identify problem in ONE sentence, give solution directly.

**Template:**
```
[Problem]? [Lösning i en mening]. ([Förtydligande i parentes])
```

**Example:**
```
Svårt att få till tydliga, mätbara Business Outcomes? Ta bort värdeord
och skriv ut vad ni faktiskt menar istället. (värdeord = enkelt / tryggt / säkert / snabbt)
```

## STRUCTURE

### 1. Opening (1-3 lines)
Use one of the style examples above. Get straight to the point.

### 2. Development (5-10 lines)

**If introducing a concept:**
- Explain briefly with definition as question + answer
- "En Stakeholder Review? Produktteam och stakeholders samlas för att..."

**If giving instruction:**
- Follow up with examples or benefits
- "Hur ser ett Business Outcome ut? Så här tycker jag: [exempel med bullets]"
- "När teamet har X händer tre saker: 🔵..."

**If showing problem:**
- Give concrete examples directly with rhetorical questions
- "Vad betyder 'enkelt'? Färre klick? Kortare implementationstid?"

### 3. Concrete Examples (3-5 lines)
- ALWAYS with numbers
- Format: "X" → "Konkret mätbar formulering från Y till Z"
- Reference workshop/project if available

### 4. Insight/Reflection (1-3 lines)
- Use "Det intressanta är att..."
- OR parentheses for bonus-insight: "(Och då kan det visa sig att...)"
- OR personal opinion: "In my humble because of experience opinion"

### 5. Ending (0-1 line)
- Simple call-to-action: "Testa!"
- OR end with reflection
- No long summary

## GENERATION INSTRUCTIONS

When generating a LinkedIn post:

1. **Identify ONE core idea** from the material you receive
2. **Choose opening style** that fits the idea (mechanism, instruction, or problem → solution)
3. **Write 10-20 lines** - no more
4. **Include AT LEAST:**
   - 1 parenthesis
   - 1 personal voice marker ("Så här tycker jag", "ack", "In my humble opinion", etc.)
   - Concrete numbers if possible
   - 1-2 rhetorical questions
5. **Avoid:**
   - Meta-observations about seeing patterns
   - Didactic headings
   - Over 20 lines
   - Multiple ideas
   - Abstract opening
   - Too much explanation

6. **Flow naturally** - not rigid structure with headings
7. **Make it actionable** - reader should be able to use this

## OUTPUT

Return only the LinkedIn post, no explanation or meta-text.
